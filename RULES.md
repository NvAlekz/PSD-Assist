# Reglas para el Agente

Antes de realizar cambios:

1. Leer PROJECT_GOAL.md
2. Leer BACKLOG.md

Proceso:

1. Seleccionar la tarea pendiente más importante.
2. Implementarla.
3. Ejecutar pruebas.
4. Corregir errores encontrados.
5. Actualizar BACKLOG.md.
6. Crear documentación necesaria.
7. Continuar con la siguiente tarea.

Prioridades:

- Funcionalidad primero.
- Estabilidad después.
- Optimización al final.

Nunca:

- Eliminar código funcional sin motivo.
- Reescribir módulos completos innecesariamente.
- Crear dependencias innecesarias.

Detenerse únicamente cuando:

- No existan tareas pendientes.
- Sea necesaria información humana.
- Exista un bloqueo técnico real.
